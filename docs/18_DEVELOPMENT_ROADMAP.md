# Success Criteria

The MVP is complete when users can

✓ Create an organization

✓ Invite team members

✓ Upload utility bills

✓ Receive AI insights

✓ Track EcoScore

✓ Generate reports

✓ Set sustainability goals

✓ Monitor progress

---

# Development Rules

Build one feature completely before starting another.

Commit frequently.

Test continuously.

Update documentation whenever architecture changes.

---

# Final Principle

Shipping a polished MVP is more valuable than building dozens of unfinished features.
